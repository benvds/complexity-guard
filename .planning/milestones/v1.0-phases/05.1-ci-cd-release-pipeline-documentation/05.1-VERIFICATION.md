**Documentation section from `docs/releasing.md` (lines 139-175):**

```markdown
## How Homebrew SHA256 Works

Here's how the SHA256 placeholder mechanism works:

- The Homebrew formula (`publication/homebrew/complexity-guard.rb`) contains 4 placeholder strings:
  - `PLACEHOLDER_SHA256_AARCH64_MACOS`
  - `PLACEHOLDER_SHA256_X86_64_MACOS`
  - `PLACEHOLDER_SHA256_AARCH64_LINUX`
  - `PLACEHOLDER_SHA256_X86_64_LINUX`

- During the `homebrew-update` CI job, the workflow:
  1. Downloads the actual binary archives created by the build job
  2. Computes their SHA256 checksums using `shasum -a 256`
  3. Replaces the placeholders with the real checksums using `sed`
  4. Updates the version number in the formula

- The placeholder naming convention matches the Zig cross-compilation target names for consistency

- After the workflow completes, the updated formula needs to be manually copied to the Homebrew tap repository (future enhancement: automate this with a personal access token)

**Why placeholders are needed:** We can't know the SHA256 checksums until after the binaries are built, and the checksums must be in the formula for Homebrew to verify download integrity.
```
